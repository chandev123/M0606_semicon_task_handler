[Algorithm R&D] 사선삽입(Diagonal Insertion) 알고리즘 모듈
요약 : 수학적 궤적 계산 및 단독모듈 구동테스트 완료
현상 : 사선틸팅(tilting) 및 정밀삽입을 위한 자체 알고리즘 연산하였으나, 메인제어 노드(task_node.py)에 통합하여 실제 하드웨어(M0609)에 인가했을 때, z축(깊이) 방향으로 약 +3mm 위치오차 발생. 로봇의 엔드이펙터가 타겟 작업대(커스텀 선반)를 기준치 이상으로 압박하여 물리적 파손(collision)이 발생하는 하드웨어 이슈 발생
원인분석 : ROS2 메인제어 노드와 로봇 제어기간의 힘 피드백(force feedback) 데이터 연동실패로 파악됨. 알고리즘은 로봇 끝단의 힘 데이터를 받아야 작동하지만, 짧은 개발기간 내에 제조사 API의 실시간 힘 데이터를 ROS2 노드로 파싱(parsing)하여 끌어오는 기능을 메인 아키텍쳐에 완벽히 통합하지 못함. 로봇은 물리적 저항을 감지하지 못하는 순수위치 제어 상태로 동작하면서 오차를 힘으로 밀어붙여 파손발생
보류 결단 : 힘 데이터를 ROS2에 안정적으로 끌어오는 인터페이스 구축에 추가 리소스를 투입할 경우, 전체 프로젝트 마감기한을 맞출 수 없다고 판단
대안 : 추가적인 하드웨어 파손 리스크를 막고, 기한 내 MVP 런칭을 위해 알고리즘 통합을 과감히 보류. 대신 로봇의 안전이 보장되는 하드코딩 API 호출방식의 절대좌표 기반 제어를 진행하여 파손리스크 0% 시퀀스 완수
